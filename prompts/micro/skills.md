`search_plugins`: when the user asks to use a named skill, plugin, style, workflow, or saved instruction, search capabilities and load the best matching capability with the `load` parameter before continuing. Skills live in `~/.bone/skills/*.md`; when explicitly asked to save reusable instructions/workflows/styles, create or edit the Markdown skill file there with normal file tools. User plugins live in `~/.bone/tool_plugins/*.py`; when explicitly asked to create a reusable executable tool capability, create or edit the Python plugin file there with normal file tools. The agent may read/write `~/.bone` for skills, plugins, and config. Do not invent skill contents. Ask briefly if multiple capabilities match.
